## 修复 Cloudflare 部署报错 (drink-repo)
- 报错：`wrangler deploy` 报 "Could not detect a directory containing static files"。
- 根因：仓库是纯静态单文件站点，但 (1) 缺少 `wrangler.toml` 告诉 wrangler 静态资源目录；(2) 首页文件名是 `healthy.html` 而非 `index.html`（git 记录显示之前删过 index.html）。
- 修复：
  1. `git mv healthy.html index.html`（根路径需要 index.html）。
  2. 新增 `wrangler.toml`：`assets = { directory = "./", not_found_handling = "404-page" }`，采用 Worker + 静态资源模式（对应部署命令 `wrangler deploy`）。
  3. 新增 `.gitignore`（忽略 .wrangler/、node_modules/）。
- 验证：`npx wrangler@4.110.0 deploy --dry-run` 通过，读取到静态文件，无报错。已本地 commit（未 push）。
- 注意：部署走 git，需 push 后 Cloudflare 才会重新构建。
- 额外发现（未处理，需用户确认）：index.html 用了大量 Tailwind 工具类（grid/text-4xl/bg-green-50 等），但没有引入 Tailwind CSS，线上排版/配色会失效。
